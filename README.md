# SMS - Simple Messaging Service (Spring Boot)

Учебный backend-проект мессенджера, разработанный в процессе изучения **Spring Boot**.  
Проект реализует REST API для регистрации пользователей, авторизации, обмена сообщениями и управления сообщениями.  
Frontend представлен простыми HTML/JS-страницами для ручного тестирования API в браузере и локальной сети.

Проект создавался поэтапно параллельно с обучающим интенсивом, но тема и архитектура были выбраны самостоятельно.

## Функциональность

### Пользователи
- Регистрация пользователя
- Авторизация (логин через `POST`, без передачи хэша пароля в URL)
- Получение информации о пользователе
- Получение списка всех пользователей
- Пагинация при запросе списка пользователей
- Получение списка контактов пользователя
- Удаление аккаунта

### Сообщения
- Отправка сообщений между пользователями
- Получение истории сообщений между двумя пользователями
- Редактирование сообщений
- Удаление сообщений

### Обработка ошибок
- Централизованный обработчик ошибок (`@ControllerAdvice`)
- Отдельный DTO для ошибок
- Корректные HTTP-коды ответов

## Архитектура проекта

- Контроллеры (`Controller`) — REST API
- DTO — отдельные объекты для передачи данных
- Сервисный слой (`Service`) с бизнес-логикой
- Централизованная обработка исключений
- Явное разделение ответственности между слоями

Проект не претендует на production-архитектуру, но соблюдает принципы читаемости и расширяемости кода.


## Безопасность

В проекте **осознанно не используется Spring Security, JWT или OAuth**.  
Аутентификация реализована в упрощённом виде, так как на момент разработки соответствующие темы не рассматривались в учебных материалах.

Пароль:
- передаётся через `POST`-запрос
- хранится в виде хэша
- не возвращается клиенту в ответах API

 

## Frontend

Frontend представляет собой **три отдельные HTML-страницы**:
- регистрация
- логин
- чат

Он предназначен для:
- ручного тестирования API
- демонстрации работы приложения
- использования в локальной сети

Frontend не является production-решением и не использует фреймворки.


## Запуск проекта

**Чтобы использовать приложение в своей локальной сети**, необходимо в файлах скриптов веб-страниц `sms\src\main\resources\static\scripts` поменять значение переменной `host_ip` в начале файла на IP-адрес в локальной сети компьютера, на котором запущено приложение. Затем нужно пересобрать архив командой `mvn clean package`. 
После этого приложение настроено и готово к работе. Его можно запустить через Docker или локально по инструкции ниже.

### Локально
```
mvn clean package -DskipTests или ./mvnw clean package -DskipTests
java -jar target/*.jar
```

### Через Docker
```
cd sms
mvn clean package -DskipTests или ./mvnw clean package -DskipTests
cd MessengerDocker
docker-compose up -d
```

_Примечание:_ При запуске приложения без Docker локально должна существовать база данных, к которой оно подключится. Для этого в `docker-compose.yml` можно закомментировать раздел `messenger`, вернуть `ports` в разделе `db` и изменить ссылку в `application.properties` на старую (она закомментирована). После этого можно запускать контейнер, в котором будет только база данных отдельно от приложения. 

Приложение будет доступно по адресу:
```http://localhost:8080```

Друге пользователи локальной сети могут подключиться по адресу:
```http://<IP-адрес_хоста_приложения_в_локальной_сети>:8080```
